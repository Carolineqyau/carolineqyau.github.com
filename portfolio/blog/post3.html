Blog Design Research

May 13th, 2020

I learned that the performance of html can be improved by preloading the link element’s rel attribute. It allows preloading resources that our pages will need soon after loading early in the lifecycle of a page load, before the browser’s main rendering machinery.

Another cool thing is that audio and video elements allow us to play audio and video media natively without external software support. I wonder if this is how our lecture videos are embedded in our assignment guidelines?

Lastly, a hint given when adapting to mobile design is to never set text using viewport units alone, since it will not allow users to zoom in on text. A solution to this is using calc() and combining vw units to fixed size units such as em or rem which will allow the text to be zoomable. For example: h1 { font-size: calc(1.5rem + 3vw); }
