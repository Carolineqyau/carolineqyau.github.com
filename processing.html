Environmental Issue

My game revolves around the issue of sea pollution. The motivation of the game is to inspire people to avoid polluting the sea including the actions of throwing man-made non decomposable materials such as plastic bottles into rivers, and to decrease the usage of non decomposable materials and adopt a sustainable lifestyle.

There are 5.25 trillion pieces of plastic waste estimated to be in our oceans. 269,000 tons float, 4 billion microfibers per km2 dwell below the surface. 70% of our debris sinks into the ocean's ecosystem, 15% floats, and 15% lands on our beaches.

Plastics take 500-1000 years to degrade; currently 79% is sent to landfills or the ocean, while only 9% is recycled, and 12% gets incinerated.

Problem Solving

1. I thought maybe it is because the keyPressed condition only works if the key is pressed, and it is not a good option for setting the state of the screens.

2. I decided to use an integer as a tracker for the screen condition. The default value set in the setup() will be the welcome screen. Only if keyPressed in the welcome screen could change the value of the screen condition integer.

3. when the screen condition integer changes value when keyPressed, the program goes into another condition where the game starts. Since the value of the screen condition integer cannot be changed after, the game remains in the game screen.

Lessons Learnt

The aesthetics of the program is easy to add on if I have a good working base code structure with simpler objects. The logic of many operations, conditionals, and aesthetics are based on the foundations that we learnt, especially the if statements. A logical and effective thought process when developing a program can be very efficient, elegant to read and easy to debug, such as setting up custom functions (void) and declaring global variables at the beginning (easy to keep track of and to edit).
