Design Brief

Problem

We have often experienced being busy dealing with urgent work and putting off what’s important in the long term. President Eisenhower once said, “I have two kinds of problems, the urgent and the important. The urgent are not important, and the important are never urgent.” I want to create an app that represents the Eisenhower Matrix, and help users to manage tasks and be efficient.

Target Audience

Individuals that have a lot going on, find it difficult to save time for long term goals and are occupied by busy work. Groups that need to divide urgent tasks to be more efficient, and have everyone work on important long term goals.

  • Needs: Find time to complete small milestones leading to the long term goal. Delegate low-priority tasks to others to complete.
  • Goals: Get a job, find time to prepare for GRE, strategic planning of the department’s long term goals.
  • Painpoints: A lot of busy work that needs to be done and procrastinated time spent on TV etc. Find it difficult to manage steps to long term goals. A lot of busy work is not important and wastes time.

Potential App Features/Tasks

Eisenhower Matrix that helps users prioritize urgent tasks and work towards important long term goals at the same time. Help users identify unimportant tasks that should be avoided or delegated.

Users can pick appropriate time slots to dedicate that time for long term goals. The app has default modes that provide pre-filled information for the user for common tasks such as GRE preparation, which skips the long haul of entering information with fewer steps.

User Research Insights

Trend

  • Use this app as a supplement to their currently existing organization apps.
  • Used for important matters, not so much for the unimportant row.

Things that people liked

  • Task divider.
  • Auto subtask generation and reminder that helps them build towards the bigger goal.

Features wanted

  • A casual notes section.
  • Chrome extension and compatibility with other organization apps.
  • Subtasks for essay.
  • More view options, like calendar time block.

Critiques

  • It is too binary to put things as important/nonimportant.
  • Standalone app will not be useful, compatibility with other calendar apps is wanted.
  • Delegate and delete tasks will not be used as often as the important row.

Usability Report

Executive Summary

I tested for 3 prototyped user tasks with 7 users, including:

    Task 1: Return to "Tasks" screen and add a new event.

    Task 2: Navigate between major homescreens of the app.

    Task 3: Add and edit details of a new event from the Calendar screen.

Upon testing the prototype with 7 users, most were confused by the two date/time related fields, one at the top (about duration as start and end date) and the other at the bottom (as time blocks used for commitment).

Task List

I asked each user to add an event at the calendar view screen named “Apply to Jobs” with 3 tasks (including Finding Job Postings, Get Referrals, Update Portfolio) to the eisenhower matrix as a “DO” priority and set the time block of “Find Job Postings” task as Mondays Wednesdays 7-8pm, and “Get Referrals” task as Thursdays Saturdays 7-8pm.

Tester Profile

The user testing demographic details are as follows:

  • Age ranges from 21 to 55, average 26 years old
  • Highly educated, most are in the process of completing or have completed a bachelor's degree
  • All have a smartphone and try out new apps regularly
  • Have a tight schedule with many tasks
  • Have had experience using a planner or some type of calendar app

Usability Problems

Common User Errors I Observed

  • Clicking the wrong area and getting confused by what a particular field is about.
  • Not being able to find the calendar screen, failing to connect the icon to the word “calendar”.
  • Pressing + button before filling out the task in the 2 default tasks list. Leaving out time blocks as empty and unfilled.
  • Not seeing the settings button and “done” button at the bottom because of not scrolling to the bottom.
  • Leaving out time blocks as empty and unfilled.

User Painpoints

  • There is not an option to just have an event list as one single day instead of a range.
  • Not sure what Eisenhower matrix is about, still pretty confused. Consider adding guides at the introduction screen to explain each functionality and make the app more accessible. Consider older audiences that may not be so tech savvy.
  • Don’t see social media or share button or send email button, not very convenient.
  • Change the calendar setting as one single line instead of two, it is confusing.
  • Can’t find where to add an event. The + button is at the top of the screen and the calendar icon is at the bottom of the screen, it is confusing.

User Feedback

About what was useful in performing the task:

  • The divisions are pretty clear.
  • Color scheme is visually pleasing, and it functionally marks the priority of each event.
  • The purple calendar title at the top beneath the overlay indicates which section of the page I am currently in, which is helpful.
  • The grey color indicates those are default fields that have not been activated or filled.
  • The colors show which fields I have edited and are done.

Usability Problems with the Design

  • Date and day settings are confusing, why are they divided into two sections?
  • Tapping one task makes it grey, unsure what that means.
  • Not sure if supposed to fill out every field or it is just selective.

Action Items

Part 2: Main Screen

Add a small line of text beneath the icons to improve usability.
Add introductory guides to assist users to learn how to use this app.
Have the eisenhower matrix introduction page be easily accessible from the main page to let users know what this is about.

Part 2: Add Event Screen

Make the design similar to Airbnb where when you tap into one field you can set the date and duration, or just have users choose when they want to commit their time at the first date setting instead of dividing it into two like Google Calendar.
Users didn’t know that you could scroll down the add event page, have an arrow or bar or animation that signifies the page is scrollable.
Keep the date and day settings consistent at one place, or have a mark that explains what they do.
Only make the must-filled fields grey and others hidden and collapsed.
